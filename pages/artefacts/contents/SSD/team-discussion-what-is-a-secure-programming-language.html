Team Discussion: What is a Secure Programming Language?


What factors determine whether a programming language is secure or not?

There are a lot empirical surveys and studies conducted in the past years on discussing security challenges, such as a survey conducted by (Fulton et al., 2021) and a study composed by (Croft et al. 2022). We list out characteristics of the secure programming language discussed from those studies.

  1. Memory Management
  2. Concurrency handling for the race condition
  3. NULL pointer
  4. Type Safety
  5. Type Checking
Could Python be classed as a secure language? Justify your answer.

Even though Python has equipped some of above characteristics, it doesn't come with type safety and type checking. Python is easy to learn and is a good fit for most of programming tasks. However, it has a weakness as it provides a dynamic typing feature. It means that Python will convert the variable to different type based on the changes of value. It becomes dangerous and vulnerable as hackers can exploit this to attack the system.

Python would be a better language to create operating systems than C. Discuss.

As Python lacks of pointer capability, type safety, and type checking. I don't think it is a good fit for creating operating systems than using C language.
