  1. What is ReDOS and what part do ‘Evil Regex’ play?

    The ReDOS is one of denial of service attacks which attackers exploit the mechanism of validating by the Regular Expression. The application will stuck at validating the input and response slowly.

    The 'Evil Regex' is a Regular Expression that containing a pattern prone to get stuck on validating the input value.

  2. What are the common problems associated with the use of regex? How can these be mitigated?

    Most likely the system uses the 'Evil Regex' pattern in both frontend and backend applications. Attackers send the crafted input value to the backend system, and compromise the backend system.

    The possible mitigation methods are:
    • Set timeout for validating the input value.
    • Validating the input value with a separate thread.
    • Avoid using the 'Evil Regex' pattern to validate the input value from clients.

  3. How and why could regex be used as part of a security solution?

    We can use the regx to perform validation on the input data before continue to proceed the request. For example, we can use them to filter out the incorrect email address, the phone number, and passport number etc. If the system find any incorrect data from the request, it will deny to handle the request, and return an error to clients.
