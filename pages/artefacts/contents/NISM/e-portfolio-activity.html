Questions for the Bhatt, D. (2018) article

  1. What does the article teach you about carrying out vulnerability scans using Kali?

    It teaches me to how to perform reconnaissance activity for the target remote machine, such as domain, sub-domain and open ports information. It also demonstrates how to compromise the remote system to acquire root access to the target system.

  2. What issues might you encounter?

    The Kali Linux only set up a default user "root" for us at the beginning. It is dangerous to perform any action on your machine. The Linux system won't check the permission for the root account. So, if we run any test with the default root account, it would damage the system and resulting in become unbootable. (Leroux, 2020).

  3. How would you overcome them?

    Runs the Kali Linux by using the virtualization technique to avoid running it as my main operation system. It isolates my main operation system from the Kali Linux. It also provides a way to persist a snapshot at anytime. So, it can be recovery to the last good condition (Bhatt, 2018).

Questions for the Bhingardeve, N. & Franklin, S. (2018) article

  1. How do their results compare with your initial evaluation?

    In the article, the evaluation results for those tools are quite simple and it doesn't elaborate much in the article. It provides simple indication for the comparison item.

  2. What do you think of their criteria?

    I think the evaluation results are not comparing those penetration tools. For example, the SQLmap is the only tool for exploiting SQL injection to the target system. I think we should compare with the similar tools to see the differences among same type of tools.

Questions for the UoEO Computing Team, 2020 statement

  1. What are the pros and cons of using Kali Linux vs. Nessus?

    Pros for using Kali Linux:
    1. Free and open source.
    2. It bundled with a verity of tools, and easy to use.
    3. Flexible. It allows to add options when using those penetrating test tools.

    Cons for using Kali Linux:
    1. Requires a high computer specification to use the virtualization technique to host it.

    Pros for using Nessus:
    1. Easy to install. It can be installed on Kali Linux.
    2. It good at performing vulnerability scanning for the target website.

    Cons for using Nessus:
    1. It is not a out-of-box tool to perform vulnerability scan by only one click. Users have to set up the test environment before performing it.

  2. Has this changed your original evaluation score?

    No. The Kali Linux is a free, easy to use, and a out-of-box tool for conducting different scanning and testing. It can install the Nessus tool to perform vulnerability scanning.
