Collaborative Discussion 2 - Alternatives to SQL


Nowadays, there have four popular NoSQL technologies in the market. They are graph databases, key/value stores, document stores, and object databases. They have different characteristics and serve various purposes.

The graph databases were designed for handling the complex relationship between entities without having costly “join” operations. It stores relationships accompanied by the entity to the graph databases. [1] It returns relationships directly while querying, searching, and finding relationships among entities.

The key/value stores were designed for accessing the data quickly. It is a set to store the value and the key in the memory. [2] It is good at frequently retrieving and persisting a small portion of consecutive data set from the in-memory storage.

The document databases were designed for storing the entire document into the database on a column-row basis. It can be considered a non-relational database which means that the document database doesn’t have to pre-define the schema for the collection. [3] It provides feasibility to store different kinds of documents in the same collection without doing data migration.

The object databases were designed for storing the entire object into the database. It is the key feature of the object database. [4] It provides capabilities to persist and parse the object from the database. So, developers don’t have to cater to the serialisation and deserialisation of the object.
