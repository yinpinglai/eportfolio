My response to Kaoru Kitamura's post

Apart from the flexibility and extensibility advantages, other merits of the NoSQL technology are scalability and elasticity.

NoSQL databases are good at scaling out. For example, MongoDB, one of the NoSQL leaders in the market, provides horizontal scaling and sharding features (MongoDB, N.D.). However, when the database grows up, we need to add up capacity to cater to all requests. Matt pointed out that the relational databases cannot scale-out, and it is costly to upgrade them vertically in a single machine (Matt A., N.D.). The NoSQL database supports scaling horizontally by adding more nodes into the cluster. In addition, the NoSQL database doesn't need powerful, expensive, complex hardware for scaling out. For example, if we need to add up to 10TB capacity to the existing cluster, we can split it into ten nodes, and each node has 1TB capacity. Consequently, it is much cheaper and more affordable for consumers.

NoSQL databases are elasticity. NoSQL databases support storing a massive volume of data set with the sharding technique by splitting them into smaller data sets (Juan P.,2022). This technique propagates those smaller data sets across different nodes in the cluster to enhance the performance of the read/write operation.

My response to Kei Yiu Yvone Chan's post

Both relational databases and NoSQL databases have their advantages and disadvantages. It is hard to judge which one is the best choice for the data warehouse. As a result, we need a logical way to decide which database technology we should use. Sara (2020) introduced a practical way to choose between the relational and NoSQL databases by examining the data set. If we need a fast performance on reading/writing and a flexible database, we should go for a NoSQL database. On the other hand, if we need analysis data and query quickly, we should go for a relational database.

References:
  • Sara A. (2020) "Databases 101: SQL vs. NoSQL: Which Fits Your Data Better?" Available from: https://towardsdatascience.com/databases-101-sql-vs-nosql-which-fits-your-data-better-45e744981351 [Accessed on 15 Feb 2022]
